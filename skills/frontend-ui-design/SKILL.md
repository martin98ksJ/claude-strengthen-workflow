## 输入

- 页面需求描述或原型截图
- 目标平台（H5 移动端 / PC 管理后台 / 响应式）
- 使用的 UI 框架（Vant / Element Plus / Ant Design 等）

## 触发条件

以下情况必须先出 UI 设计文档再写代码：
- 新页面开发（全新路由）
- 现有页面大改版（布局/交互大幅变更）
- 涉及 3+ 组件的复合页面
- 用户明确要求先设计

以下情况不需要：
- 单组件微调（文案/颜色/间距）
- bug 修复
- 已有设计文档且无变更

## 职责（必须做）

### 1. 页面概述

- [ ] 页面名称、路由路径、所属模块
- [ ] 页面目标（用户在此页面完成什么任务）
- [ ] 页面入口（从哪些页面/操作进入）
- [ ] 数据来源（调用哪些 API）

### 2. ASCII 线框图

用 ASCII art 画出页面结构，标注区域名称：

```
┌─────────────────────────────┐
│         NavBar              │
│  ← 返回    页面标题    ···   │
├─────────────────────────────┤
│                             │
│   [区域A: 搜索/筛选栏]       │
│                             │
├─────────────────────────────┤
│                             │
│   [区域B: 主内容区]          │
│   ┌───────────────────┐     │
│   │  列表项/卡片/表单   │    │
│   └───────────────────┘     │
│                             │
├─────────────────────────────┤
│   [区域C: 底部操作栏]        │
│   [ 取消 ]    [ 确认 ]      │
└─────────────────────────────┘
```

规则：
- 每个可交互区域用 `[区域名]` 标注
- 按钮用 `[ 按钮文字 ]` 表示
- 输入框用 `[____输入提示____]` 表示
- 图标用文字描述 `(icon:name)` 表示
- Tab 切换用 `| Tab1 | Tab2 | Tab3 |` 表示
- 列表项画 2-3 个代表，用 `...` 表示更多
- 弹窗/浮层单独画一个线框图

### 3. 组件清单

表格列出每个区域使用的组件：

| 区域 | 组件 | Props/配置 | 说明 |
|------|------|-----------|------|
| NavBar | van-nav-bar | title, left-arrow | 顶部导航 |
| 列表 | van-list + van-pull-refresh | loading, finished | 分页加载 |
| 列表项 | van-cell | title, value, label | 每行数据 |

### 4. 状态与数据

- [ ] 列出页面所有响应式状态（ref/reactive）
- [ ] 标注数据来源（API / route params / store / 本地计算）
- [ ] 标注加载时机（onMounted / watch / 用户触发）

```
状态清单：
- loading: boolean          ← API 请求状态
- list: Item[]              ← GET /api/items
- searchKeyword: string     ← 用户输入
- currentPage: number       ← 分页控制
- finished: boolean         ← 是否加载完全部
```

### 5. 交互说明

用编号列出所有用户交互场景：

```
交互流程：
1. 进入页面 → 显示骨架屏 → 加载首页数据 → 渲染列表
2. 下拉刷新 → 重置分页 → 重新加载
3. 滚动到底 → 加载下一页 → 追加到列表
4. 点击列表项 → 跳转详情页 /detail/:id
5. 点击筛选 → 弹出筛选面板 → 选择后重新加载
6. 数据为空 → 显示空态插图 + 引导文案
7. 加载失败 → 显示错误提示 + 重试按钮
```

### 6. 四态处理

每个数据区域必须说明四种状态的展示方式：

| 状态 | 展示 |
|------|------|
| Loading | 骨架屏 / Spinner / 按钮 loading |
| 空态 | 插图 + 引导文案（如"暂无数据"） |
| 错误态 | 错误提示 + 重试按钮 |
| 正常态 | 数据列表/表单/卡片 |

### 7. i18n Key 清单

列出页面所有需要翻译的文案，按区域分组，中英文对照：

```
i18n keys:
  pageName:
    title: "页面标题" / "Page Title"
    searchPlaceholder: "搜索提示" / "Search..."
    filterAll: "全部" / "All"
    filterActive: "进行中" / "Active"
    emptyText: "暂无数据" / "No data"
    emptyAction: "去创建" / "Create now"
    errorText: "加载失败" / "Load failed"
    retryBtn: "重试" / "Retry"
    loadingMore: "加载中..." / "Loading..."
    noMore: "没有更多了" / "No more data"
```

### 8. 响应式断点（如适用）

- 移动端（≤ 768px）：默认设计目标
- 平板（768-1024px）：需要特殊处理时说明
- 桌面（≥ 1024px）：B 端必须，C 端按需

（表格）

## 边界（不做什么）

- 不涉及视觉设计细节（具体颜色值、字号、圆角 → 按 UI 框架主题）
- 不涉及代码实现（组件写法、hook 封装 → 见 vue/react-conventions）
- 不涉及后端 API 设计（接口定义 → 见 db-api-design）
- 不涉及动效/动画细节（过渡时长、缓动曲线 → 按框架默认或设计稿）
- 不替代 Figma/Sketch 等专业设计工具（ASCII 线框图用于开发对齐，非交付设计稿）

## 输出

设计文档存放 `design/` 目录，命名 `ui-<模块>-<页面>.md`。
文档结构：概述 → 线框图 → 组件清单 → 状态数据 → 交互流程 → 四态处理 → i18n Keys → 注意事项。
